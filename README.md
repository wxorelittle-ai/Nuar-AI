# РестоПульс — Competitor Scraper Agent

Первый рабочий модуль AI-платформы **РестоПульс** для ресторана **Nuar** (Тюмень).

Агент раз в неделю собирает открытые данные по конкурентам (2ГИС, Яндекс.Карты, VK,
городские СМИ), сравнивает их с прошлой неделей и присылает управляющему
форматированный разведдайджест в Telegram.

---

## Что делает агент

1. **Сбор** — по списку конкурентов из [`config/competitors.yaml`](config/competitors.yaml)
   собирает: рейтинг, число отзывов, свежие отзывы (2ГИС/Яндекс), посты и число
   подписчиков (VK), упоминания в городских СМИ.
2. **Сравнение** — сопоставляет свежий снимок с предыдущим (хранится в PostgreSQL
   или локальном JSON-файле), вычисляет изменения рейтингов и активности.
3. **Дайджест** — формирует сообщение в Telegram Markdown и отправляет управляющему.

Каждый снимок сохраняется — на нём строятся понедельные сравнения и история метрик.

## Веб-система МЭТР (приватный backend)

Полноценная система управляющего: онбординг + мгновенный анализ, **настройки
AI-ассистентов** и сценарии на их основе.

```bash
cd restopulse
pip install -r requirements.txt
python -m web.app                 # или: uvicorn web.app:app --port 8000
# открыть http://localhost:8000        — анализ
# открыть http://localhost:8000/settings — настройки AI-ассистентов
```

**Экран анализа** (`/`): вводите ресторан → профиль, конкуренты в округе,
позиция, SWOT, приоритеты, контент-план и вечерняя сводка «Слово МЭТРА».
Без `YANDEX_API_KEY` — демо-данные округа (помечено плашкой); с ключом — реальные
данные через Yandex Geosearch API.

**Настройки AI-ассистентов** (`/settings`): подключение моделей, на которых
работает МЭТР. Поддержаны:

| Провайдер | Что нужно |
|---|---|
| **YandexGPT** | API-ключ сервисного аккаунта + Folder ID |
| **GigaChat** (Сбер) | Authorization key (Base64); при SSL-ошибке — выключить проверку сертификата |
| **OpenAI-совместимый** | Ключ + базовый URL (OpenAI, DeepSeek, OpenRouter, локальные) |
| **Claude** (Anthropic) | API-ключ |

Кнопка «Проверить связь» делает короткий реальный вызов. Блок «Мэтр отвечает»
генерирует черновик ответа на отзыв на активной модели.

**Где хранятся ключи.** В `data/settings.json` на вашем сервере (в git не
коммитится). Наружу и в API-ответы ключи не отдаются — только маска вида `…1234`.
API-ключи нельзя размещать в публичной статической витрине (`site/`) — они
доступны только приватному backend.

Стратегический движок (SWOT, приоритеты, контент-линии, «Слово МЭТРА»
rule-based) работает всегда, без AI-ключей — на введённом профиле и метриках.

### Контент и публикации (`/content`)

Контент-агент пишет посты на активной AI-модели и публикует их в соцсети.
Поток: **генерация черновика → правка/утверждение человеком → публикация**.
Публикуется только то, что утверждено (принцип бренда — ручное подтверждение).

- **Сети**: ВКонтакте (на стену сообщества) и Telegram (в канал).
- **Подключение каналов** — на `/settings`, раздел «Каналы публикации»:
  - VK: ключ доступа сообщества (права «Стена» + «Управление») и числовой id сообщества;
  - Telegram: токен бота (@BotFather, бот — админ канала) и канал (`@username` или `chat_id`).
- **Автопубликация**: у поста можно задать время; утверждённые посты с наступившим
  сроком публикует планировщик (`agents/orchestrator/scheduler.py`, проверка раз в минуту).

**Модерация** («Мэтр проверяет», `agents/moderation/`) — обязательный шлюз перед
публикацией, работает без AI-ключей:
- **блокирует** ценовое давление (скидки, акции, проценты, «бесплатно») — по
  бренд-гайду для премиум-сегмента это запрещено — и тексты сверх лимита сети;
- **предупреждает** об эмодзи, восклицаниях, капсе, слишком коротком тексте.

Заблокированный пост не уходит в соцсеть; на кнопке «Проверить» замечания видны
заранее. Опционально можно включить AI-проверку тона активной моделью (совет, не блок).

**Идеи от разведки** (`agents/content/ideas.py`) — замыкают петлю
*мониторинг → идея → пост*. На странице контента показывается панель тем:
- **ответные идеи** на сигналы конкурентов (активность в VK, упоминания в СМИ) —
  берутся из последних снимков разведки; напр. «Арбат активно постит → анонсируйте
  свой вечер»;
- **базовые идеи** по контент-линиям бренда — есть всегда.

Клик по идее подставляет сеть, контент-линию и тему в композер — дальше генерация,
модерация, публикация. Идеи становятся точнее после запуска еженедельной разведки.

Токены каналов, как и ключи ассистентов, хранятся в `data/settings.json` на вашем
сервере, маскируются в ответах API и не попадают в публичную витрину.

### Тренды — «Мэтр в тренде» (`/trends`)

Динамика интереса к темам (блюда, кухни, форматы) за ~2 месяца по официальному
**Wikipedia Pageviews API** (`agents/trends/`) — без ключей, с любого IP. Считает
рост/падение просмотров статей, делит на восходящие/снижающиеся/стабильные, и по
растущим темам предлагает идеи для контента и сезонного меню. Темы можно задать
свои (по умолчанию — подборка для ресторана). Это прокси интереса, не прямые продажи.

### VIP CRM — «Мэтр помнит» (`/crm`)

Гостевая база и персональные касания (`agents/crm/`): загрузка **CSV** (гибкое
распознавание заголовков рус/англ, разделители `,`/`;`) → **сегментация**
(VIP-постоянные / постоянные / корпоративные / разовые с потенциалом / разовые) →
**триггеры**: день рождения (за 3 дня) и «давно не было» (>45 дней). По триггеру
МЭТР готовит **личное сообщение от метрдотеля** (без скидок; шаблон, а при
подключённом ассистенте — персонализированный на модели). Сообщения —
**черновики для ручной отправки** гостю. Данные гостей хранятся на сервере (152-ФЗ).

### Соцсети конкурентов — «Мэтр наблюдает» (`/social`)

Углублённый анализ VK-сообщества конкурента (`agents/social/`): по названию группы —
подписчики, **ритм постинга** (постов/неделю), **вовлечённость** (средние лайки/
комментарии/просмотры, ER = лайки/просмотры), **лучший день недели**, **темы**
(частые слова) и **топ-посты по лайкам**. Нужен `VK_SERVICE_TOKEN` (dev.vk.com);
без него — понятная подсказка вместо выдуманных цифр.

### Рекрутинг — «Мэтр нанимает» (`/recruiting`)

Анализ рынка труда по данным **HeadHunter**: по ролям ресторана (официант, повар,
бармен, хостес, администратор, шеф) — число вакансий, зарплата «на руки»
(медиана и диапазон p25–p75) и **кто из заведений активно нанимает** (сигнал
расширения или текучки у соседей).

> HH закрыл анонимный поиск вакансий (отвечает 403). Чтобы работал анализ:
> зарегистрируйте бесплатное приложение на [dev.hh.ru](https://dev.hh.ru/admin)
> и задайте `HH_CLIENT_ID` / `HH_CLIENT_SECRET` в `.env`. На собственном сервере
> доступ часто открыт и без ключей — система пробует анонимно, а при 403 честно
> подсказывает, что нужно. Поиск резюме (платный, аккаунт работодателя) — следующий шаг.

### Автоматизация (оркестратор)

`python -m agents.orchestrator.scheduler` — фоновые задачи:
- еженедельный разведдайджест по конкурентам → Telegram управляющему;
- автопубликация запланированных утверждённых постов.

## Быстрый старт (локально, без Docker)

```bash
cd restopulse
python -m venv .venv
.venv\Scripts\activate            # Windows
# source .venv/bin/activate       # Linux/macOS

pip install -r requirements.txt

copy .env.example .env            # Windows (cp на Linux/macOS)
# заполните TELEGRAM_BOT_TOKEN, TELEGRAM_CHAT_ID, VK_SERVICE_TOKEN
```

Разовый прогон (собрать данные и прислать дайджест):

```bash
# сухой прогон — печатает дайджест в консоль, ничего не отправляет и не пишет в БД
python -m agents.competitor_scraper.scraper --dry-run

# боевой разовый прогон — соберёт, сохранит снимок, отправит в Telegram
python -m agents.competitor_scraper.scraper --once
```

Запуск по расписанию (каждый понедельник 09:00 МСК):

```bash
python -m agents.orchestrator.scheduler
```

Без PostgreSQL модуль работает на локальном файле снимков
(`data/snapshots.json`) — этого достаточно, чтобы уже видеть недельные изменения.
Для продакшена включите PostgreSQL (переменная `DATABASE_URL`).

## Production — запуск на сервере (Docker)

Полноценный стек: **PostgreSQL + веб-панель (gunicorn) + фоновый воркер + Caddy**
(HTTPS автоматически). Пошаговая инструкция для рег.облака / Ubuntu — в
[DEPLOY.md](DEPLOY.md).

```bash
cp .env.example .env
#  ОБЯЗАТЕЛЬНО задайте:
#    ADMIN_PASSWORD    — пароль входа в панель
#    SECRET_KEY        — длинная случайная строка (openssl rand -hex 32)
#    POSTGRES_PASSWORD — пароль БД
#    METR_SITE_ADDRESS — домен (авто-HTTPS) или :80 (доступ по IP)
#  токены ассистентов/каналов можно задать позже через /settings

docker compose up -d --build      # db + web + worker + caddy
docker compose logs -f web        # логи веб-панели
# панель: https://<домен>  (или http://<IP> при METR_SITE_ADDRESS=:80)
```

- **HTTPS**: Caddy сам получает и продлевает сертификат Let's Encrypt, если в
  `METR_SITE_ADDRESS` указан домен (A-запись → IP сервера, порты 80/443 открыты).
  Порт приложения 8000 наружу не публикуется — только через Caddy.
- **Хранилище**: при заданном `DATABASE_URL` (в compose задаётся автоматически)
  все данные — посты, настройки, снимки — в PostgreSQL, том `pgdata` на вашем
  сервере (152-ФЗ). Без `DATABASE_URL` — локальный режим на JSON.
- **Безопасность**: панель за паролем (`ADMIN_PASSWORD`), сессия — подписанная
  cookie. Без пароля вход открыт — только для локальной разработки.
- **Здоровье**: `GET /health` → `{"status":"ok","storage":"postgres"}`.
- **Надёжность публикаций**: неудачная автопубликация повторяется (до 3 раз),
  затем пост помечается статусом «Ошибка».

## Источники данных и их особенности

| Источник | Метод | Нужен ключ | Надёжность |
|---|---|---|---|
| **VK** | Официальный VK API (`wall.get`, `groups.getById`) | Да — сервисный токен | Высокая |
| **2ГИС** | Публичная карточка фирмы + встроенный JSON | Нет (best-effort) | Средняя* |
| **Яндекс.Карты** | Официальный Search/Places API либо парсинг карточки | Для API — да | Средняя* |
| **Городские СМИ** | Поиск по 1tmn.ru, nashgorod.ru | Нет | Средняя |

\* 2ГИС и Яндекс активно защищаются от ботов. Для стабильной работы в проде
рекомендуется подключить официальные бизнес-API (2GIS Business API,
Yandex Geosearch/Places API) — заготовки под них есть в парсерах. HTML-парсинг
используется как fallback и при блокировке аккуратно отдаёт `ok=False`, а не
выдумывает данные.

**Этика краулинга:** соблюдаются `robots.txt`, задаётся User-Agent, между
запросами выдерживается пауза (`REQUEST_DELAY_SEC`), ретраи с экспоненциальной
задержкой. Никакого агрессивного параллельного краулинга.

## Соответствие 152-ФЗ

Модуль собирает **только открытые обезличенные данные** о заведениях-конкурентах
(рейтинги, публичные посты, публичные отзывы). Персональных данных гостей Nuar
здесь нет. При переходе к Этапу 2 (VIP CRM) вся гостевая база должна храниться
на серверах в РФ (Cloud.ru / VK Cloud).

## Переменные окружения

См. [`.env.example`](.env.example). Ключевые:

- `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID` — куда слать дайджест.
- `VK_SERVICE_TOKEN` — сервисный ключ доступа сообщества/приложения VK.
- `DATABASE_URL` — строка подключения PostgreSQL (пусто → JSON-fallback).
- `YANDEX_API_KEY` — ключ Yandex Search/Places API (опционально).
- `DGIS_API_KEY` — ключ 2GIS Catalog/Business API (опционально).
- `REQUEST_DELAY_SEC` — пауза между запросами (по умолчанию 2.0).
