# МЭТР / MAÎTRE — статическая витрина (для размещения на сайте)

Один самодостаточный файл — [`index.html`](index.html). Внутри уже всё:
разметка, стили, движок анализа на JavaScript и полностью применённый
**фирменный стиль МЭТР v1.0**. Сервер не нужен, движок стратегии (SWOT,
приоритеты, контент-план, «Слово МЭТРА») работает прямо в браузере.

Это **демо-витрина** на примере ресторана Nuar (Тюмень): посетитель называет
заведение → МЭТР осматривает округу, показывает профиль, конкурентов, SWOT,
приоритеты и выдаёт вечернюю сводку в своём голосе. Показатели округа
иллюстративные (о чём честно сказано в форме).

## Фирменный стиль (применено)

- **Название**: МЭТР / MAÎTRE. Позиционирование — «AI-управляющий», не «платформа».
- **Знак-бабочка** (галстук метрдотеля) — в шапке, лоадере, футере и favicon.
- **Палитра**: графит `#14171A`, латунь `#C9A227`, бутылочный `#0F2B25`
  (раздел «наблюдения»/конкуренты), скатерть, серебро. Одна латунь на экран,
  без теней и градиентов.
- **Шрифты**: Playfair Display (голос МЭТРА, слоганы, названия конкурентов
  курсивом), Inter (интерфейс), JetBrains Mono (все числа, рейтинги, метрики).
  **Встроены прямо в файл** (base64, наборы латиница + кириллица) — сайт работает
  полностью автономно, без единого внешнего запроса и без интернета для шрифтов.
- **Нейминг модулей**: «Мэтр наблюдает / помнит / отвечает / приглашает».
- **Голос**: первое лицо, факт → рекомендация со сроком → человек в конце.
  Без эмодзи, без восклицаний, без канцелярита.

## Импорт (встраивание) на существующий сайт

Витрина — это цельная HTML-страница, поэтому вставляется на сайт через
**`<iframe>`**. Она сама сообщает свою высоту, и блок подстраивается под
содержимое (форма → дашборд) без полос прокрутки.

**Шаг 1.** Залейте `index.html` куда-нибудь, откуда он открывается по ссылке
(тот же хостинг, где стоит сайт, отдельная папка, Netlify и т.п.). Запомните
адрес — например `https://ваш-домен/metr/index.html`.

**Шаг 2.** В нужное место сайта вставьте HTML-блок (Tilda — блок «T123 / HTML»;
WordPress — блок «Произвольный HTML»; Wix/Битрикс — «Вставка HTML»):

```html
<iframe id="metr-frame" src="https://ваш-домен/metr/index.html"
        title="МЭТР" scrolling="no"
        style="width:100%;border:0;display:block;min-height:640px"></iframe>
<script>
  addEventListener('message', function (e) {
    if (e.data && e.data.type === 'metr-height') {
      var f = document.getElementById('metr-frame');
      if (f) f.style.height = e.data.height + 'px';
    }
  });
</script>
```

Замените `src` на свой адрес. Готовый пример — в файле
[`embed.html`](embed.html) (можно открыть локально и посмотреть, как это
выглядит; там `src="index.html"` рядом).

> Если конструктор не разрешает вставлять `<script>` — оставьте только `<iframe>`
> и задайте фиксированную высоту (например `min-height:1600px`); авто-подстройка
> тогда не сработает, но страница будет видна целиком.

## Как разместить отдельной страницей

Файл статический — подойдёт любой способ:

- **Обычный хостинг / VPS**: залейте `index.html` в корень сайта (или в папку
  `/demo/`) по FTP/панели. Откроется по адресу домена.
- **Встраивание (Tilda, WordPress, Битрикс)**: как отдельная страница/файл со
  ссылкой, либо через `<iframe src="index.html">`.
- **GitHub Pages / Netlify / Cloudflare Pages**: положить файл и включить
  статический хостинг.
- **Проверить локально**: открыть `index.html` двойным кликом.

> Файл самодостаточный: шрифты, стили и логика внутри. Внешних запросов нет —
> работает офлайн и на любом хостинге без настройки.

## Что можно поправить под себя (всё в одном файле)

- **Палитра/токены** — CSS-переменные в начале `<style>` (`--brass`, `--graphite`…).
- **Демо-цифры** — функции `demoProfile()` и `demoCompetitors()` в `<script>`.
- **Текст сводки МЭТРА** — функция `maitreNote()`.
- **OG-теги/домен** — блок `<head>` для превью в мессенджерах.
- **Форма заявки/контакт** — сейчас витрина без формы захвата; добавляется блоком
  в конце (по запросу).

## Связь с рабочей версией

Логика анализа здесь — точный порт серверного движка
(`agents/analyzer/strategy.py`). В боевом МЭТР тот же движок работает на
**реальных данных** с карт и соцсетей (Yandex Geosearch API, 2ГИС, VK) —
витрина показывает ровно тот результат, что клиент получит на своих данных.
